# Автоматизированное развертывание и настройка Neovim 

В данном репозитории находится скрипт `install_nvim.sh`, предназначенный для быстрой и чистой установки редактора Neovim (v0.12.2) из официальных бинарных релизов GitHub, развертывания структуры конфигурации на базе Lua и предварительной инициализации менеджера плагинов `lazy.nvim`.

---

## Архитектура и этапы работы скрипта `install_nvim.sh`

Скрипт разбит на логические шаги, обеспечивающие предсказуемое состояние системы:

1. **Обновление системы и системные зависимости:**
   * Синхронизирует кэш `apt` и устанавливает инструменты сборки (`build-essential`), утилиту системного буфера обмена (`wl-clipboard` для Wayland), системный рекурсивный поисковик (`ripgrep`), а также `nodejs` и `npm` (необходимы для работы некоторых LSP-серверов).
2. **Чистая установка Neovim v0.12.2:**
   * Принудительно удаляет старые бинарники, скачивает официальный архив `nvim-linux-x86_64.tar.gz`, распаковывает его в изолированную директорию `/opt/nvim-linux-x86_64` и создает глобальный симлинк в `/usr/local/bin/nvim`.
3. **Окружение Rust & Cargo:**
   * Проверяет наличие `cargo`. Если пакет отсутствует, устанавливает тулчейн Rust через официальный скрипт `rustup.rs` и инициализирует переменные окружения.
4. **Установка `cargo-binstall`:**
   * Разворачивает утилиту для быстрой установки прекомпилированных бинарных крейтов Rust, минуя долгий шаг локальной компиляции.
5. **Установка `tree-sitter-cli`:**
   * Устанавливает CLI-интерфейс компилятора Tree-sitter через `cargo binstall`. Это необходимо для корректной сборки локальных парсеров синтаксиса.
6. **Создание структуры директорий:**
   * Инициализирует стандартную XDG-структуру папок для Neovim: `~/.config/nvim/lua/config/` и `~/.config/nvim/lua/plugins/`.
7. **Генерация конфигурационных файлов (Контекст Lua):**
   * `init.lua`: Задает базовые настройки системы, маппинги кастомных типов файлов (Ansible, Terraform, Docker, Helm), подключает общий системный клипборд (`unnamedplus`) и отключает легаси-провайдеры (Perl, Ruby).
   * `lua/config/lazy.lua`: Автоматически скачивает (bootstrap) менеджер плагинов `lazy.nvim` стабильной ветки и настраивает лидер-клавиши (`<Space>`).
   * `lua/plugins/hop.lua`: Конфигурирует плагин `hop.nvim` для быстрого клавиатурного перемещения по коду по лейблам.
   * `lua/plugins/treesitter.lua`: Настраивает абстрактное синтаксическое дерево (AST) подсветки кода, автокоманды для сброса старого синтаксиса Vim, а также инжекшены для парсинга Bash-скриптов внутри инструкций `RUN` в `Dockerfile`.
   * `lua/plugins/mason-lsp.lua`: Инициализирует менеджер внешних зависимостей `mason.nvim`, интегратор `nvim-lspconfig` для подключения серверов к буферам и движок автодополнения `nvim-cmp` (с поддержкой сниппетов `LuaSnip`, путей и буферов).
8. **Фоновая синхронизация плагинов:**
   * Запускает Neovim в фоновом режиме (`--headless`) и выполняет команду `Lazy! sync`. Это гарантирует, что при первом интерактивном запуске пользователя все плагины уже будут склонированы с GitHub в директорию `~/.local/share/nvim/lazy/`.

## Порядок ручной инициализации окружения

После успешного завершения работы скрипта необходимо запустить Neovim интерактивно и применить две команды для окончательной сборки рантайма.

### 1. Компиляция парсеров синтаксиса (Treesitter)
Встроенная абстрактная подсветка требует компиляции Си-бинарников парсеров под конкретную архитектуру. Выполните команду внутри Neovim:

```vim
:TSInstall yaml dockerfile bash lua markdown markdown_inline python

```

* **Зачем это нужно:** Обычный регулярный синтаксис Vim не умеет глубоко разбирать структуру файлов. Treesitter строит синтаксическое дерево, что обеспечивает идеальную подсветку кода, правильную работу фолдинга (сворачивания блоков через `foldexpr`) и контекстные инжекшены (например, подсветка Bash внутри Dockerfile).

### 2. Развертывание бинарников языковых серверов (Mason)

Для работы статического анализа, автодополнения и прыжков по коду необходимо установить бинарные файлы LSP-серверов. Выполните команду:

```vim
:MasonInstall dockerfile-language-server docker-compose-language-service ansible-language-server terraform-ls yaml-language-server bash-language-server lua-language-server

```

**Описание устанавливаемых серверов:**

* `dockerfile-language-server` (`dockerls`) — валидация инструкций Dockerfile, автодополнение директив.
* `docker-compose-language-service` — автодополнение ключей конфигурации манифестов Compose, проверка версий спецификаций.
* `ansible-language-server` (`ansiblels`) — интеграция с ansible-lint, валидация модулей, плейбуков и тасок.
* `terraform-ls` — разбор конфигураций HCL, валидация провайдеров и переменных `.tfvars`.
* `yaml-language-server` (`yamlls`) — базовый сервер для YAML-файлов с автоматической интеграцией удаленного каталога SchemaStore (проверка синтаксиса Kubernetes, GitHub Actions, GitLab CI).
* `bash-language-server` (`bashls`) — статический анализ shell-скриптов, проверка переменных, интеграция с shellcheck.
* `lua-language-server` (`lua_ls`) — разбор конфигурации самого Neovim, автодополнение встроенного API (`vim.*`), контроль глобальных переменных.

## Диагностика и обслуживание

### Полный сброс окружения Neovim (С чистого листа)

Если конфигурация или бинарные файлы серверов были повреждены, систему можно вернуть в исходное состояние одной командой:

```bash
rm -rf ~/.config/nvim ~/.local/share/nvim ~/.cache/nvim ~/.local/state/nvim

```

* `~/.config/nvim` — пользовательские файлы конфигурации.
* `~/.local/share/nvim` — склонированные плагины и установленные бинарники Mason/Treesitter.
* `~/.cache/nvim` — временные файлы кэша компиляции и логи (LSP/плагинов).
* `~/.local/state/nvim` — история сессий, undo-файлы, история команд (ShaDa).
